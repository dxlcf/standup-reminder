# 久坐提醒软件设计文档

## 1. 产品定位

一款 Windows 11 轻量级久坐提醒工具。用户下载一个 exe 文件即可运行，无需安装任何运行时环境或依赖。软件在工作时间内定时弹出置顶窗口，提醒用户起身休息，并通过倒计时和评分机制帮助用户养成习惯。

**核心约束：**

- 交付物为单个 exe 文件，大小不超过 50MB
- 零安装、零依赖，双击即用
- 数据纯本地存储，仅保留最近 30 天记录
- 不联网、不采集隐私

---

## 2. 功能设计

### 2.1 工作时间与提醒

用户配置若干工作时间段，软件仅在这些时间段内触发提醒。

| 配置项     | 默认值                         | 说明                     |
| ---------- | ------------------------------ | ------------------------ |
| 工作时间段 | 09:00–12:00、13:00–18:00       | 支持多个时间段，覆盖午休 |
| 提醒间隔   | 60 分钟                        | 每隔多久提醒一次         |
| 休息时长   | 5 分钟                         | 每次建议休息时长         |
| 延后时间   | 10 分钟（可选 5 / 10 / 15）    | 每次提醒仅可延后一次     |

提醒时间从每个时间段的起始时间加上一个提醒间隔开始计算。例如时间段 09:00–12:00、间隔 60 分钟，则提醒时间为 10:00、11:00、12:00。

### 2.2 提醒弹窗

提醒触发时弹出置顶窗口，要求如下：

- 居中显示于鼠标所在屏幕（多显示器时优先活跃屏幕，否则主显示器）
- 窗口高度 = 屏幕可用高度 × 50%，宽度 = min(720px, 屏幕可用宽度 × 45%)，最小 480 × 360
- 置顶显示，禁用标题栏关闭按钮、Esc 和 Alt+F4
- 不做系统级锁屏，不阻止 Ctrl+Alt+Delete 或任务管理器

窗口内容：

```
┌──────────────────────────────────┐
│                                  │
│       该起来活动一下了           │
│    建议站起来、喝水、伸展或远眺  │
│                                  │
│            04:59                 │
│                                  │
│     完成休息可获得 100 分        │
│                                  │
│   [ 延后提醒 ]   [ 结束休息 ]    │
│                                  │
└──────────────────────────────────┘
```

### 2.3 休息倒计时

弹窗后立即开始倒计时。倒计时期间窗口保持置顶，显示剩余时间和鼓励文案。

- 倒计时自然结束：记录 100 分，自动关闭窗口
- 用户点击"结束休息"：弹出二次确认（"才休息了 X 分钟，再坐会儿呗？"，按钮为"继续休息"和"直接开工"），确认后按比例计分并关闭

### 2.4 延后提醒

用户点击"延后提醒"：关闭弹窗，等待延后时间后再次弹出。

- 每次提醒只能延后一次，延后后再次弹窗时不再显示"延后提醒"按钮
- 延后本身不产生分数，最终得分在用户实际休息后计算

### 2.5 计分规则

单次得分公式：

```
score = round(实际休息秒数 / 计划休息秒数 × 100)
```

得分范围 0–100，保留整数。每日总分为当天所有提醒得分的算术平均值。

延后后最终完成休息的分数归属于原计划提醒时间段。

## 3. 页面设计

整个软件由四个视图组成：主界面（仪表盘）、设置页面、提醒弹窗、记录页面。主界面、设置页面、记录页面通过左侧导航栏切换，提醒弹窗为独立弹出窗口。

### 3.1 整体导航

窗口左侧固定一个窄边栏（约 56px 宽），包含：

- 顶部：应用图标
- 导航按钮（纵向排列）：主界面图标、记录图标、设置图标
- 当前选中页面的按钮高亮显示

右侧为内容区域，根据导航选中状态切换显示对应页面。边栏与内容区之间以细线分隔。

### 3.2 主界面（仪表盘）

主界面专注于展示当前状态，不包含任何设置项。布局从上到下分为四个区域：

**状态栏：** 左侧显示当前工作状态胶囊（"工作中"带绿色圆点 / "非工作时间"带灰色圆点），右侧显示当前所处时段文字（如"下午时段 13:00 – 18:00"），非工作时间时显示下一个工作时段的起始时间。

**倒计时环：** 页面中央放置一个环形进度指示器（直径约 180px），环形进度条显示当前提醒周期已过去的比例。环下方依次显示：

- "距离下次提醒"提示文字（小字、浅色）
- 剩余时间（大号数字，如"42:18"）
- 预计提醒时刻（小字，如"预计 14:00 提醒"）

非工作时间时，倒计时区域显示"当前为非工作时间"或距下一个工作时段的等待时间。

**今日数据卡片：** 三个并排的统计卡片：

| 卡片       | 内容                   |
| ---------- | ---------------------- |
| 今日得分   | 当天所有提醒的平均分   |
| 已提醒     | 今日已弹出提醒的次数   |
| 完成率     | 完整休息次数 / 已提醒次数 |

**快捷操作：** 两个并排按钮：

- "立即休息"（主按钮，绿色）：立即触发一次提醒弹窗
- "暂停 30 分钟"（次按钮，描边样式）：暂停提醒 30 分钟

### 3.3 设置页面

设置页面独立于主界面，通过侧边栏齿轮图标进入。页面顶部标题为"设置"，右上角放置"测试提醒"按钮。

设置项分为两个卡片分组：

**卡片一：工作时间段**

- 列表展示所有已配置的工作时间段，每条显示起止时间和备注标签（如"上午""下午"）
- 每条右侧有删除按钮（×）
- 列表末尾有"+ 添加时间段"按钮，点击后展开起止时间选择器
- 编辑时间段：点击已有条目可修改起止时间

**卡片二：提醒规则**

- 提醒间隔：显示当前值（如"60 分钟"），点击可编辑
- 休息时长：显示当前值（如"5 分钟"），点击可编辑
- 延后时间：显示当前值（如"10 分钟"），点击可在 5 / 10 / 15 分钟之间选择

页面底部右对齐放置"保存设置"按钮。保存后立即重新计算下次提醒时间，进行中的倒计时不中断，历史记录不受影响。

### 3.4 提醒弹窗

独立弹出窗口，不受侧边栏导航控制。详见 2.2 节。

### 3.5 记录页面

通过侧边栏列表图标进入。页面布局从上到下：

**页面头部：** 左侧标题"休息记录"，右侧为日期导航器（左箭头、当前日期文字、右箭头），可在最近 30 天内逐日切换。当天显示为"YYYY-MM-DD 今天"。

**统计栏：** 四个并排卡片：

| 卡片       | 内容               |
| ---------- | ------------------ |
| 今日总分   | 当天平均分         |
| 提醒次数   | 当天计划提醒总数   |
| 完成次数   | 完整休息的次数     |
| 延后次数   | 使用延后的次数     |

**记录列表：** 表格形式，每条记录包含以下列：

| 列名     | 说明                           |
| -------- | ------------------------------ |
| 计划     | 原计划提醒时间（如 10:00）     |
| 实际     | 实际弹窗时间（如 10:10）       |
| 时长     | 实际休息时长（如 4:00）        |
| 状态     | 完成 / 提前结束（标签样式）    |
| 延后     | 是否使用了延后（标签样式）     |
| 得分     | 本次得分（颜色区分高低分）     |

不做复杂图表，仅提供列表视图。

## 4. 状态流转

单次提醒的完整生命周期：

```
等待中(waiting)
   │
   ▼ 到达提醒时间
弹窗中(popup) ──→ 倒计时中(resting)
                    │
          ┌─────────┼─────────┐
          ▼         ▼         ▼
       延后(deferred)  提前结束    自然结束
          │       (early_done)  (completed)
     等待延后时间
          │
          ▼
     再次弹窗(不可再延后)
          │
     ┌────┴────┐
     ▼         ▼
  提前结束   自然结束
```

全局状态还包括：`outside_work_time`（不在工作时间内）和 `paused`（用户手动暂停）。

## 5. 边界情况

| 场景               | 处理方式                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| 电脑休眠后恢复     | 检查当前是否在工作时间内，是则立即弹出一次提醒，否则不弹出               |
| 软件被关闭后重启   | 读取上次状态，在工作时间内重新计算下次提醒；不补弹多个错过的提醒         |
| 多显示器           | 弹窗居中于鼠标所在屏幕，尺寸按该屏幕可用区域计算                         |
| 独占全屏应用       | 窗口设为 TopMost 尽量置顶；若被遮挡仍记录提醒已触发                      |
| 用户修改设置       | 保存后立即重新计算下次提醒时间；进行中的倒计时不中断；历史记录不受影响   |

## 6. 托盘菜单

软件启动后常驻系统托盘，菜单项：

- 打开主界面
- 立即休息
- 暂停提醒 30 分钟
- 今日记录
- 退出（二次确认："退出后将不再提醒休息，确认退出吗？"）

## 7. 技术方案

### 7.1 技术栈选择

```
语言：    C#
框架：    WPF (.NET 8)
打包：    dotnet publish 单文件发布 + ReadyToRun + Trimming
数据存储：JSON 文件
```

选择理由：

- WPF 原生支持窗口置顶、托盘图标、模态弹窗、多显示器感知，无需额外依赖
- .NET 8 单文件发布 + Trimming 后，一个 WPF 应用通常在 15–30MB 之间，满足 50MB 限制
- 发布命令参考：`dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true -p:PublishTrimmed=true -p:EnableCompressionInSingleFile=true`

### 7.2 数据存储

不使用数据库。所有数据以 JSON 文件形式存储在 `%APPDATA%/StandUpReminder/` 目录下。

文件结构：

```
%APPDATA%/StandUpReminder/
├── settings.json          # 用户设置
├── state.json             # 运行时状态（下次提醒时间、当前状态等）
└── records/
    ├── 2026-06-18.json    # 按天存储提醒记录
    ├── 2026-06-17.json
    └── ...
```

**settings.json** 结构：

```json
{
  "workPeriods": [
    { "start": "09:00", "end": "12:00" },
    { "start": "13:00", "end": "18:00" }
  ],
  "intervalMinutes": 60,
  "breakMinutes": 5,
  "deferMinutes": 10
}
```

**每日记录文件**（如 `2026-06-18.json`）结构：

```json
{
  "date": "2026-06-18",
  "sessions": [
    {
      "originalDueTime": "10:00",
      "actualPopupTime": "10:00",
      "plannedRestSeconds": 300,
      "actualRestSeconds": 300,
      "score": 100,
      "status": "completed",
      "deferred": false
    },
    {
      "originalDueTime": "11:00",
      "actualPopupTime": "11:10",
      "plannedRestSeconds": 300,
      "actualRestSeconds": 240,
      "score": 80,
      "status": "early_done",
      "deferred": true
    }
  ]
}
```

**自动清理：** 软件启动时扫描 `records/` 目录，删除 30 天前的记录文件。每天一个 JSON 文件，一个月最多 30 个文件，每个文件通常不超过几 KB，总存储量可忽略不计。

**state.json** 结构：

```json
{
  "nextReminderTime": "2026-06-18T14:00:00",
  "currentStatus": "waiting",
  "lastActiveDate": "2026-06-18"
}
```

用于软件重启后恢复状态。

### 7.3 模块划分

| 模块             | 职责                                     |
| ---------------- | ---------------------------------------- |
| SettingsManager  | 读写 settings.json                       |
| Scheduler        | 根据设置计算提醒时间，每 10 秒轮询检查   |
| ReminderWindow   | 提醒弹窗 UI 和交互逻辑                   |
| ScoreCalculator  | 计算单次得分和每日平均分                 |
| RecordStore      | 读写每日 JSON 记录，执行 30 天自动清理   |
| StateManager     | 管理 state.json，处理休眠/重启恢复       |
| TrayManager      | 系统托盘图标和右键菜单                   |
| NavigationManager| 侧边栏导航与页面切换逻辑                 |

### 7.4 定时器设计

后台使用 DispatcherTimer 每 10 秒检查一次：

1. 当前时间是否在某个工作时间段内？
2. 是否已到达或超过下次提醒时间？
3. 当前是否已有弹窗显示？

三个条件同时满足时弹出提醒窗口。不依赖长间隔单次 Timer，因为休眠、系统时间变更等会导致偏差。

## 8. 验收标准

### 8.1 基础功能

- 可配置多个工作时间段、提醒间隔、休息时长
- 工作时间内按间隔准确弹出提醒，非工作时间不弹出
- 弹窗置顶居中，不可通过关闭按钮 / Esc / Alt+F4 关闭
- 倒计时结束自动记录 100 分并关闭
- 提前结束按比例计分
- 延后提醒正常工作（仅限一次），延后后再次弹窗不可再延后
- 每日总分正确统计
- 托盘常驻，菜单功能正常
- 单个 exe 在 Windows 11 上双击即可运行，大小 < 50MB

### 8.2 页面导航

- 侧边栏三个导航按钮可正常切换页面，当前页面按钮高亮
- 主界面不包含任何设置项，仅展示状态和快捷操作
- 设置页面修改后保存，主界面数据实时刷新
- 各页面切换流畅，无闪烁或布局跳动

### 8.3 计分验收

测试条件：休息时长 = 5 分钟。

| 操作                | 预期得分 |
| ------------------- | -------- |
| 完整休息 5 分钟     | 100      |
| 休息 4 分钟后结束   | 80       |
| 休息 2.5 分钟后结束 | 50       |
| 休息 1 分钟后结束   | 20       |
| 立即结束            | 0        |
| 延后 1 次后完整休息 | 100      |

## 9. 不做的事情

以下内容明确排除在第一版范围之外：

- 云同步、联网、账号系统
- 复杂图表和数据可视化
- 开机自启（可后续加）
- 数据导出（JSON 文件本身即可读）
- 深色模式
- 自定义提醒音效和文案
- 番茄钟模式
- 键盘鼠标空闲检测
- 成就系统
